Industry Challenge
The Industrial Knowledge Crisis & High-Stakes Risks
Modern processing facilities struggle with fragmented documentation, slow incident resolution, and strict regulatory compliance.
⚠️ Unstructured Industrial Data Silos
  • Disparate Technical Records: Critical knowledge is trapped across OEM manuals, paper SOPs, legacy work orders, inspection logs, and incident reports.
  • Information Retrieval Delays: Engineers spend 30-45 minutes searching manuals during critical alarms or equipment trips.
  • Loss of Institutional Knowledge: Retiring technicians leave gaps in failure diagnostic procedures and equipment nuances.
🚫 The Generic AI Hallucination Trap
  • Dangerous AI Assumptions: Standard LLMs hallucinate specific operational parameters (e.g., torque specs, pressure limits) when context is missing.
  • False entity matching: Simple vector search inflates scores for common asset tags (e.g. "PMP-302"), returning irrelevant passages that lead to erroneous answers.
  • Regulatory Non-Compliance: Unverified AI advice in oil & gas / chemical facilities can cause catastrophic safety events and massive compliance fines.

Core Engine Deep Dive
Dual-Layer Hallucination-Gated RAG Copilot
How the system eliminates ungrounded LLM answers using mathematical scoring and strict backend gates.
Layer 1: TF-IDF Weighted Retrieval

Standard term-overlap vector retrieval inflates scores when an asset tag (e.g. PMP-302) appears in multiple documents. TF-IDF down-weights high-frequency terms and boosts rare technical terms (e.g., vibration limit, torque spec).

score = sum( TF(term) * IDF(term) ) / sqrt( ChunkLength ) where IDF(term) = ln( 1 + N / DocFreq(term) )

✓ Prevents irrelevant documents from clearing the relevance threshold.

Layer 2: LLM Groundedness Self-Gate

Even if a document matches topically, the LLM is instructed to explicitly evaluate if the chunk contains the exact requested fact. If grounded: false, the backend overrides the response with a mandatory safety refusal.

{ "grounded": false, "answer": "", "citations": [] } --> Backend Overrides to Standard Refusal Message

✓ Enforces zero-hallucination policy even if LLM attempts conversational guess.

Technical Specs
Technology Stack & Live API Endpoints
Modular, resilient architecture powered by Node.js, Express, Gemini 2.5 Flash, Groq Fallback, and Vanilla JS UI.
Technology Architecture Stack
  • Backend Runtime: Node.js with Express.js framework, CORS, Dotenv.
  • Primary LLM Engine: Google Gemini 2.5 Flash API (Strict JSON mode enabled).
  • Secondary LLM Fallback: Groq Llama-3.3-70B API for zero-downtime resilience.
  • Offline Standalone Cache: Pre-seeded fallback dataset for disconnected operations.
  • Frontend Stack: HTML5, CSS3 Custom Properties (Dark Glassmorphism), Vanilla JS (ES6 modules), Cytoscape.js graph rendering.
Production REST API Endpoints
Endpoint Method Description
/api/copilot POST Grounded RAG query handler with citations
/api/ingest GET / POST Fetch or trigger full graph rebuild from corpus
/api/rca POST Generate 5-Whys and Fishbone for incident
/api/compliance GET Run OISD-STD-189 telemetry audit rules
/api/compliance/resolve POST Simulate one-click remediation of violations
